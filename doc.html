🔗 Linking EasyC++ to Your Project

We recommend using CMake — it's clean, cross-platform, and flexible. You have two options:

🛠 Option 1: Link Prebuilt Binaries


cmake_minimum_required(VERSION 3.30)
project("For_example")

include_directories("path_to_include_dir_in_your_project/include")
link_directories("path_to_lib_dir_in_your_project/folder_with_your_OS/")

add_executable(For_example
   src/main.cpp
)

target_link_libraries(For_example PUBLIC EasyC)

🧩 Option 2: Add EasyC++ as Subdirectory


cmake_minimum_required(VERSION 3.30)
project("For_example")

add_subdirectory(EasyC)

add_executable(For_example
   src/main.cpp
)

target_link_libraries(For_example PUBLIC EasyC)
⚠️ No prebuilt binaries for macOS yet. If you're on macOS, compile the library using CMake manually. Need help? Reach out.

🚀 Program Entry

start() — Entry point of your program. Everything begins here.

🧠 Functions

  • func — Function without return
  • func_int — Returns an integer
  • func_float — Returns a decimal
  • func_string — Returns text
  • func_bool — Returns true/false

🖨 Output & Input

  • print(msg) — Print message to terminal
  • error(msg) — Show error message
  • input() — Wait for keyboard input

📦 Variables

  • decimal — Floating-point number
  • integer — Whole number
  • text — String
  • boolean — True or false

✅ Program Status

  • done — Marks successful completion
  • problem [code] — Ends program with error code

⏱ Timing & Terminal

  • wait(s) — Delay in seconds
  • clear_screen — Clears terminal
  • pause_system — Pauses terminal

📚 Arrays & Maps

  • list(type) — Dynamic array
  • map(key, value) — Key-value storage
  • add_to(container, item) — Add item to end
  • size_of(container) — Get number of elements

🗂 File System

  • create_folder(path, name) — Create folder
  • create_file(path, name) — Create file
  • delete_folder(path, name) — Delete folder
  • delete_file(path, name) — Delete file
